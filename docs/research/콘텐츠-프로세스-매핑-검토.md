## 2. 단계별 상세 매핑

### Stage 1 · 소재·리서치 ✅

리서치 문서 설계와 대시보드 구현이 가장 잘 맞는 영역.

| 리서치 문서 의도 | 대시보드 구현체 |
|----------------|---------------|
| RSS 트렌드 수집 | n8n W03 `rss-topic-collect` → `/api/rss-topics` · `/api/rss-trending` |
| AI 인사이트 정제 | n8n W10 `ai-insights` → 트렌딩·인사이트 뷰 |
| 주제 후보 선별 | n8n W09 `topic-suggest` → 주제 선별 AI 뷰 |
| 경쟁 채널 벤치마크 | YouTube vs.Avg · Outlier 태깅(W02) · 채널 분석기 |
| 레퍼런스 URL 분석 | `content-analyzer` (YouTube URL → Gemini 직접 시청 분석) |
| Naver·Tistory 리서치 | W04 `naver-blog-collect` · W05 `tistory-collect` |

**갭**: Obsidian 소재 아카이브는 대시보드 밖(수동). 프롬프트·실패 로그 체계 없음.

---

### Stage 2 · 대본·구조화 ✅

리서치 문서는 "Claude로 대본 작성"을 설계했지만 대시보드는 Gemini를 사용. 결과는 동일한 역할 분담.

| 리서치 문서 의도 | 대시보드 구현체 |
|----------------|---------------|
| 스크립트 생성 (롱폼 8분) | n8n W08 `longform-script` 1순위 · `/api/content-generate` 폴백 |
| 콘텐츠 유형 분기 | `GuideCategory`: `writing`(블로그) · `video`(숏폼) · `image`(카드뉴스) |
| 인텐트 분기 | `longform_video` · `shortform_video` · `blog` · `carousel` · `general` |
| 가이드라인 프롬프트 | `guidelines/contents_guideline.md` → Gemini 프롬프트로 직접 사용 |
| 숏폼 카테고리별 규칙 | `shortform-categories.ts` — 8개 빌트인 + 커스텀 카테고리 지원 |
| 출력 구조 | `title` · `summary` · `fullContent` · `platformVariants`(블로그) |

---

### Stage 3 · 비주얼 생성 ⚠️

콘텐츠 유형에 따라 커버리지가 크게 다름.

#### 주식 리포트 → 완전 자동

```
시세 수집(W11) → /api/stock-report · stock-report-focus
  └─ stock-chart-render.ts → ECharts → @napi-rs/canvas → PNG 파일 저장
       └─ 차트 4종 자동 생성 (가격·거래량·등락률·추세 시나리오)
  └─ content-output-html → HTML 미리보기(차트 이미지 임베드·라이트박스)
```

→ **완전 자동화. 리서치 문서 "비주얼 생성" 단계가 대시보드에서 가장 완성된 영역.**

#### 숏폼 영상 → 씬 스크립트 생성까지

```
Gemini 생성 → fullContent (시간대별 씬 스크립트)
  └─ 캡컷 등 편집 도구에서 클립 연결·자막 삽입 → 실제 영상 완성 (외부)
```

→ **대시보드는 씬 스크립트(대본·화면 묘사·자막·편집 메모)까지 생성. 실제 영상 편집은 캡컷 등 외부 도구.**

#### 일반 블로그·카드뉴스

```
Gemini 생성 → 본문 중 📷 이미지 가이드 블록 삽입
  └─ "직접 제작·삽입" 안내 카드 (실제 이미지 미생성)
```

### Stage 4~6 · 음성·자막·편집 ❌

대시보드 커버 없음. 전량 외부 도구.

| 단계 | 리서치 문서 도구 | 비고 |
|------|----------------|------|
| 음성 나레이션 | 본인 목소리 · Bark(MIT) · Chatterbox | 마이크 환경 별도 필요 |
| 자막 | Whisper / whisper.cpp | M1 Mac 로컬 실행 권장 |
| 편집·조립 | FFmpeg · MoviePy · Remotion | 수동 3~5편 후 자동화 검토 |

---

### Stage 7 · 썸네일 ❌

대시보드에서 `📷 이미지 가이드 블록`이 삽입되지만 썸네일 전용 가이드는 없음. 블로그 본문용 이미지 가이드와 동일한 형식으로 처리됨.

**리서치 문서 식별 갭과 일치**: `구조적 빈틈` 항목에도 "썸네일 시스템 — 규칙 없음(⚠️)"으로 명시됨.

---

### Stage 8 · 배포·멀티유즈 ⚠️

"생성→변환"까지는 대시보드가 처리. "업로드"는 전부 수동.

| 기능 | 구현 상태 |
|------|---------|
| 3플랫폼 메타 자동 생성 | ✅ `blog-platform-variants.ts` — Naver·Tistory·Blogger 제목·태그·메타 동시 생성 |
| 발행용 HTML 미리보기 | ✅ `content-output-html` — 차트 이미지·라이트박스·색상 포함 완성 HTML |
| 원소스 멀티유즈 변환 | ✅ `content-studio` 뷰 (포맷 변환 UI) |
| 숏폼→블로그 파생 구조 | ✅ fullContent(블로그·숏폼 씬 스크립트) 동시 생성 |
| Naver 실제 업로드 | ❌ 공식 포스팅 API 없음 — 수동 |
| Tistory 실제 업로드 | ❌ Open API 신규 등록 중단 — 수동 |
| Blogger 자동 발행 | ❌ Blogger API v3 존재하나 미연동 |
| YouTube 업로드 자동화 | ❌ 미구현 |

---

### Stage 9 · 성과 측정 ✅

리서치 문서 STEP 5("발행·데이터 관찰")에 해당. 대시보드가 핵심 분석 인프라 역할.

| 측정 항목 | 대시보드 구현체 |
|----------|--------------|
| YouTube 조회수·vs.Avg·Tier | W01 수집 → `videos` 테이블 → vs.Avg 계산 |
| 아웃라이어 탐지 | W02 `outlier-tagging` → `outlier_tags` |
| 레퍼런스 채널 벤치마크 | 채널 등록 → Shorts/롱폼 분리 vs.Avg |
| Naver 블로그 조회수 | W07 `naver-blog-views` |
| Tistory 성과 수집 | W05 `tistory-collect` |
| 경쟁 채널 URL 분석 | `content-analyzer` (Gemini 직접 시청) |

---

### Stage 10 · 과정 녹화 ❌

리서치 문서 "2번 트랙(비하인드)"의 원재료. 대시보드 범위 밖.  
`production-tracker`(제작 진행 보드)가 진행 상황을 추적하지만, 화면 녹화 자체는 OBS/Cap 등 외부.

## 3. 전략적 수렴 포인트

리서치 문서의 전략 의도와 대시보드 구현이 일치하는 핵심 지점.

### 3-1. 원소스 멀티유즈 — 연구 전략 = 대시보드 설계

리서치 문서에서 최우선 수익화 아이템(🥇)으로 꼽은 **"B-2 원소스 멀티유즈 변환기"** 가 대시보드에 이미 구현됨.

```
콘텐츠 생성 1회 →
  fullContent       → 블로그 본문 or 숏폼 씬 스크립트
  platformVariants  → 3플랫폼 제목·태그·메타 자동 분기
  content-output-html → 발행 전 완성 미리보기 HTML
```

→ **"채널·콘텐츠·제품이 한 점으로 수렴"** 이라는 연구 결론이 대시보드에서 실현 중.

---

### 3-2. 주식 주제 — 미결 → 사실상 결정

연구 문서 12절 "우주 vs 주식 미결"이지만, 대시보드 개발 투자는 명확히 주식으로 수렴.

| 연구 문서 판단 | 대시보드 현황 |
|--------------|-------------|
| "주식 = 성향상 유리, 데이터가 분석 도구 재료" | W11 시세 수집 → W12 자동 리포트 파이프라인 완성 |
| "다양한 주제 + 주식 시황 리포트" | 일일 리포트·포커스 리포트·섹터 분석 3모드 운영 |
| "쌓는 데이터가 분석 도구 씨앗" | `generation_history` + `videos` + `stock_*` 테이블 누적 중 |

→ **"주식 시황" 주제 확정으로 연구 문서 미결 의사결정 종결 가능.**

---

### 3-3. 벤치마크 분석 = 대시보드 핵심 기능

연구 문서 STEP 1(벤치마크 채널 분석)이 대시보드의 핵심 MVP.

- 리서치 문서 분석 템플릿 항목(구독자·vs.Avg·터진 영상·후크 방식) → 대시보드 채널 등록 후 자동 계산
- 벤치마크 5개 채우기(수동) → 대시보드 vs.Avg + Outlier 태깅으로 대체 가능

---

### 3-4. 비진정성 정책 회피 = 목소리 해설 → 대시보드 미반영

연구 문서에서 강조한 **"목소리 해설 = 비진정성(AI 슬롭) 정책 회피 장치"** 는 대시보드가 다루지 않는 영역(4단계 음성·나레이션). 대시보드는 "쓸 것을 만들어주는 도구"지 "말하는 도구"가 아님. 역할 분담 명확.

## 4. 갭 및 통합 기회

### 4-1. 단기 통합 가능 (난이도 낮음)

| 갭 | 현황 | 통합 방안 |
|----|------|---------|
| **성과 로그 → 분석 도구 순환** | `generation_history`와 YouTube `videos` 데이터가 미연결 | 발행 이력에 YouTube video_id 연결 → 성과 추적 자동화 |
| **썸네일 가이드 블록 추가** | 현재 `📷 이미지 가이드`만, 썸네일 전용 없음 | `contents_guideline.md`에 썸네일용 가이드 블록 포맷 추가 |
| **주식 주제 미결 의사결정 닫기** | 연구 문서에 "미결"로 남음 | 연구 문서(`종합기록-마스터.md`) 12절 업데이트 |

### 4-2. 중기 통합 검토 (중간 난이도)

| 갭 | 내용 |
|----|------|
| **영상 편집 결과물 연결** | 캡컷에서 편집 완료 후 영상 파일 경로/URL을 `production-tracker`에 기록하면 1→10 파이프라인 가시화 |
| **Blogger API 자동 발행** | 연구 문서 배포 자동화 목표. `Blogger API v3` 존재, OAuth 연동만 추가하면 3플랫폼 중 1개 자동화 가능 |
| **제휴 링크 자동 삽입** | ElevenLabs(22% 반복 수수료)·캡컷 등 제휴 툴 링크를 `fullContent` 하단에 자동 삽입하는 규칙 |

### 4-3. 대시보드 범위 밖 (계속 외부)

| 단계 | 이유 |
|------|------|
| 음성·나레이션(4단계) | "목소리 해설 = 진정성"이므로 자동화하면 오히려 정책 위반 소지 |
| 자막 생성(5단계) | Whisper 로컬 실행이 M1에서 빠르고 무료 → 별도 연동 ROI 낮음 |
| 편집·조립(6단계) | 수동 3~5편 후 반복 부분만 FFmpeg 스크립트화 (리서치 문서 전략 그대로) |
| 과정 녹화(10단계) | OBS 외부, 대시보드가 개입할 영역 없음 |

## 5. 커버리지 요약

```
대시보드가 잘 하는 것         대시보드 밖
─────────────────────────    ──────────────────
소재 발굴·트렌딩 (Stage 1)    음성·나레이션 (Stage 4)
대본·구조화·가이드 (Stage 2)   자막 생성 (Stage 5)
주식 차트 PNG 자동 생성 (3)   영상 편집·조립 (Stage 6)
씬 스크립트 생성 (3 부분)     썸네일 제작 (Stage 7)
원소스 멀티유즈 변환 (Stage 8) 과정 녹화 (Stage 10)
성과 측정·분석 (Stage 9)
```

**대시보드 정체성**: "콘텐츠 인텔리전스 + 텍스트·데이터 생성 플랫폼"  
**대시보드 밖 영역**: 영상·음성 제작 (OBS · 마이크 · 캡컷 등 편집 도구)

두 영역은 겹치지 않으므로 현재 역할 분담이 적절함.  
통합 우선순위는 **성과 데이터 순환**(발행 이력 ↔ YouTube 성과 연결)이 장기 "채널→제품" 수렴의 핵심 연결고리.
