## 1. 나 — 성향분석: "분석형 빌더 (Data-Driven Builder)"

6개 문항 응답을 종합한 진단.

| 항목 | 응답 | 해석 |
|------|------|------|
| 동기 1순위 | 데이터·전략 분석 | 숫자·로직 파고드는 게 핵심 동력 |
| 동기 2순위 | 제품 직접 만들기 | 분석한 걸 직접 구현하고 싶어함 |
| 동기 3·4순위 | 마케팅 / 고객 응대 | 사람 상대보다 시스템 상대가 편함 |
| 수익 스타일 | 안정적 부수입이면 만족 | 한 방보다 꾸준히 쌓는 형 |
| 고객 선호 | 상관없음, 돈되면 OK | 고객군은 유연 |
| 투입 시간 | 주 5~10시간 | 손 많이 가는 모델은 부적합 |
| 홍보 태도 | 콘텐츠로 자연스럽게 | 직접 영업 ❌, 콘텐츠 유입 ✓ |
| 운영 방식 | 꾸준히 손보며 키우기 | 데이터 보며 개선하는 능동 운영 |

**한 줄 요약**: 데이터·전략을 깊게 파서 직접 만드는 건 좋지만, 영업은 콘텐츠로 대신하고 싶고, 고객 응대 부담 없이 꾸준히 개선하며 안정 부수입을 쌓고 싶은 빌더.

## 2. 주제를 좁혀온 과정

1. **수익화 아이템 카테고리**에서 출발(개발자 툴·크리에이터 툴·금융·생활·마케팅형).
2. 금융(투자 백테스트)이 성향엔 맞았으나, **"비전문 도메인을 AI 이해에만 기대 만드는 위험"** 을 스스로 인지하고 걸러냄.
3. 그래서 기준에 **"도메인 친숙도"(내가 이미 사용자인 영역)** 를 최상위로 추가 → "scratch your own itch".
4. 본인이 내부자인 6개 영역(개발/콘텐츠/AI학습/육아/영어/지역) 중, 성향 4필터(분석·콘텐츠홍보·응대부담↓·재구독)로 교차 분석.
5. **콘텐츠 제작 워크플로우**가 최적으로 도출.
6. 이후 시드가 "AI 영상을 만드는 과정·결과물 자체가 콘텐츠"로 확정됨.

### 도메인 × 성향 교차표
| 도메인 | 분석 | 콘텐츠홍보 | 응대부담↓ | 재구독 | 종합 |
|--------|:---:|:---:|:---:|:---:|------|
| 콘텐츠 제작 워크플로우 | ✓ | ✓ | ✓ | ✓ | 🥇 |
| 개발자 생산성·도구 | ✓ | △ | ✓ | ✓ | 🥈 |
| 영어 학습 | △ | △ | △ | ✓ | 중 |
| AI 학습·강의 소화 | △ | ✓ | △ | △ | 중 |
| 육아·자녀 교육 | ✗ | △ | ✗ | △ | 보류 |
| 지역·생활 니즈 | ✗ | ✗ | △ | ✗ | 보류 |

## 3. 확정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| **시드** | AI 영상 생성 콘텐츠 (만드는 과정·결과물이 콘텐츠) | 코딩 아님. 만드는 대상이 곧 주인공 |
| **2트랙** | 1번=결과물(간판) / 2번=과정(비하인드) | 2번은 1번 만들 때 따라오는 부산물 |
| **전략** | 채널 우선 (제품 아님) | 제품 시장은 레드오션, 채널은 자산으로 남음 |
| **시그니처** | "AI로 만든다" 콘셉트 고정 | 주제는 열되 콘셉트로 일관성 (다양성+일관성) |
| **노출** | 목소리 해설만 (얼굴 X) | 카메라·조명·실사편집 제거 + 진정성·정책·신뢰 확보 |
| **언어** | 한국어 | 영어권은 포화·고RPM, 한국어권은 빈자리 |

## 4. 리서치 ① 수익화 구조

### 주제별 RPM (조회 1,000회당 실수령, 유튜브 45% 수수료 제외 후)
| 주제 | RPM | 비고 |
|------|-----|------|
| 신용카드·금융 | $18~45 | 최고가 |
| 금융·테크 | $8~40 | 보수적 $8~15 |
| 교육·지식 | $9~14 | |
| 애니/트루크라임 | $8~13 | |
| 게임 컴필 | $2~4 | 최저 |

**원리**: 광고주가 그 시청자를 얼마나 원하느냐(경쟁·국가·구매임박도)로 결정. → 고RPM 주제(테크·교육)를 의도적으로 섞으면 같은 조회수로 더 번다.

### 롱폼 vs 숏폼 (격차 거대)
| 구분 | RPM | 100만 조회 시 |
|------|-----|---------------|
| 롱폼(금융) | $9~21 | $9,000~21,000 |
| 숏폼 | $0.03~0.10 | $50~100 |
→ 숏폼=유입, 롱폼=수익 기둥.

### 페이스리스 특성 ⚠️
- 브랜드 협찬·멤버십·강의 수익이 인물 채널보다 약해 **총수익 30~50% 낮음** (단 제작비 70%↓라 순이익 비슷).
- 권장: 협찬보다 **고RPM 주제 + 제휴**.
- 키묘님 강점: "목소리 해설 + 2번 트랙(과정)"이 페이스리스의 약점(강의 수익)을 부분적으로 되살림.

### 타임라인·비용
- 수익화 조건: 구독 1,000 + 시청 4,000시간. 잘 돼도 6~12개월.
- 1~3개월 미미 → 3~6개월 월 $100~500 → 6~12개월 탄력.

## 5. 리서치 ② 유튜브 AI 콘텐츠 정책 — "AI는 환영, 기만은 금지"

### 두 개의 분리된 정책
| 정책 | 무엇 | 위반 시 |
|------|------|--------|
| ① 합성 콘텐츠 표시 | 투명성(오인 방지) | 미표시 제재 |
| ② 비진정성 콘텐츠 | 수익화 자격(독창성) | 수익화 박탈 |

### ① 표시 의무
- 필요: 실존 인물 AI 음성 클론·딥페이크·실제 사건 합성 장면 (Studio에서 토글).
- 불필요: 대본·편집·썸네일 보조, AI 음악, 명백한 비현실 창작.
- **표시는 순위·수익화 불이익 없음.** "숨기면 문제, 표시하면 무해." (2025-05-21 전면 시행)

### ② 비진정성 콘텐츠 (= 'AI 슬롭' 규제) ★
- 2025-07-15 개칭, 양산형 기준 강화. 2026-01 단속 웨이브.
- **2026년 탐지는 채널 전체를 평가** → 동일 포맷 반복·사람 편집 지문 없는 합성 내레이션 적발.
- 제대로 표시해도 사람 창의성 없으면 ②로 제재 가능.

### 키묘님 시사점
- "AI로 만든다" 콘셉트는 합법, 표시는 떳떳이.
- **목소리 해설 + 관점 = 비진정성 회피의 정답.**
- "다양한 주제"는 format variety로 유리하나, 일관성은 "동일 템플릿"이 아니라 "목소리·관점·룩"으로 줄 것.

## 6. 리서치 ③ Higgsfield 툴 역량 — "시네마틱엔 강하나 만능 아님"

### 강점
- Cinema Studio 3.5 카메라 시뮬레이션(70+ 프리셋), 포토리얼 비주얼.
- 멀티모델 한 구독(Sora 2·Veo 3.1·Kling 3.0·Seedance 2.0·Nano Banana Pro 등).
- Soul ID(캐릭터 일관성), Marketing Studio, **MCP 연동**(Claude Code와 자동화 가능).

### ⚠️ 한계
- **클립 10~20초 한계** → 8분 롱폼은 클립 수십 개 이어붙이기.
- **크레딧이 진짜 비용**: 프리미엄(Sora·Veo) 클립당 40~70, 저가(Kling) ~6. 반복 3~5회 감안 실단가 Kling $0.61~1.03/편.
- **캐릭터 드리프트**(다중 샷 얼굴 흔들림) → 동일 캐릭터 전략 어려움.
- 결정적: **내레이션 중심 설명형엔 주력 도구 아님** → 비주얼 생성기일 뿐, 완성은 별도.

### 시사점
- 역할 분담: Higgsfield(비주얼) / Claude(대본) / 본인 목소리(나레이션) / 편집도구(조립).
- 시그니처는 "캐릭터"보다 **"룩·콘셉트"** 로 → 키묘님 선택과 일치.
- 룩은 **저가 모델 기준**으로 설계해 크레딧 절약.

## 7. 벤치마크 (사례 + 분석 템플릿)

### 한국어 AI 채널 사례
- **진도 설이** — 진돗개를 사람처럼 AI 구현. 일관 캐릭터, 음식 롱폼 평균 3만+.
- **문명스토리** — 문명사·역사·철학 AI 내레이션. 페이스리스 지식형(목소리 해설과 가장 가까움).
- **인공능지** — AI 풍자·밈 숏폼. 유머 기획력.
- (영어 참고) **Curious Refuge**(AI 필름 교육=2번 트랙 정석), **Sabrina Ramonov**(AI 실무 교육).

**교훈**: 성공 공통점은 기술이 아니라 "무엇을 누구에게"의 기획력. AI는 대체자가 아니라 확장자.
**빈틈**: "다양한 주제를 AI로 만들되 과정을 목소리 해설로 함께 보여주는" 채널은 비어 있음 → 키묘님 자리.

### 분석 템플릿 (채널 5개 채울 항목)
채널명 / 구독자 / 7포맷 중 무엇 / 시그니처 룩 / 주제 범위 / 일관성 축 / 업로드 주기 / 롱숏 비중 / 터진 영상 / 후크 방식 / 본인 노출 / 수익모델 / **약점·빈틈**
→ 채운 뒤 3질문: ① 공통 성공 패턴 ② 빈틈 ③ 나의 차별 각도.

### 7가지 지배 포맷
역사 1인칭 POV / AI 단편·시네마틱 / 일관 캐릭터 세계관 / 페이스리스 교육 / AI 자동화·비즈니스 / AI 아트 / AI 뉴스·툴.

## 10. 영상 편집툴 직접 개발 검토

- **풀 편집기(CapCut형)**: ❌ 비추천 (수년·무료대안 많음).
- **자동 조립 파이프라인**: ✅ 가능 + 강점 부합. 클립 stitching→나레이션→자막→출력을 코드로.
- ⚠️ **함정**: 영상 만들기 전 도구부터 만들지 말 것. 수동 3~5편 → 반복 부분 식별 → 그것만 자동화.
- ★ **수렴**: 그 자동화 파이프라인 = 2번 트랙 콘텐츠 = 원래 원했던 제품.
- 스택: FFmpeg + MoviePy(파이썬) 또는 Remotion(React, 프론트 경험 적합) + Whisper.

## 12. 콘텐츠 주제 검토 (우주 vs 주식)

- **A(과정 메타) vs B(실제 콘텐츠)는 충돌 아님** — B(본편)를 만들어야 A(비하인드)가 생김 = 이미 정한 1번/2번 트랙 그대로.
- 진짜 질문은 **"B의 주제를 뭘로?"**
- **우주 교양**: 공개 이미지(NASA 등)+AI 영상, 시네마틱 "보는 맛". 늦게 터지나 오래감. 분석 재미는 적음.
- **주식 시황**: 데이터·차트·시의성. 매주 갱신 = 능동 운영과 맞고, 쌓는 데이터가 분석 도구의 재료. **성향상 유리**.
  - ⚠️ 단 금융 규제 → "이 종목 사세요"❌, "이번 주 시장은 이랬다" 시황·교양 선까지만.
- 트래블튜브식 "공개 이미지+AI영상" 레퍼런스는 두 주제 다 적용 가능.
- 결정 기준: "내가 분석하는 재미" 우선 → 주식 / "많이 모이는 것" 우선 → 우주.
- **한 채널 한 주제 집중** (둘 섞으면 알고리즘·시청자 혼란).

## 13. 수익 다각화 7갈래 + 수익 가지도

### 7갈래 요약
| 갈래 | 수익 잠재 | 노동·응대 | 성향 적합 | 시점 |
|------|:---:|:---:|:---:|------|
| 1. 디지털 상품(템플릿·프롬프트팩) | 중 | 낮음 | ★★★★★ | 채널 자리잡으면 |
| 2. 강의(자체 판매) | 중상 | 중 | ★★★★ | 채널 큰 뒤 |
| 3. 제휴(반복 수수료) | 중 | 매우낮음 | ★★★★ | 채널 자리잡으면 |
| 4. 유료 뉴스레터 | 낮~중 | 중 | ★★★ | 선택 |
| 5. 자동화 대행 | 높음 | **매우높음** | ★ | 비추천 |
| 6. 마이크로 SaaS | 높음 | 중 | ★★★★ | 장기 |
| 7. 데이터 분석 도구 | 높음 | 중 | ★★★★★ | 장기 |

**세부 근거**
- 디지털 상품: 마진 90%+, 번들/볼트가 핵심(단품 $5 함정). n8n 템플릿 $29~299+.
- 강의: 클래스101 수수료 ~40%·제작 전부 크리에이터 부담 → 자체 판매가 유리.
- 제휴: ElevenLabs 22% 12개월 반복. **Higgsfield는 제휴 없음**(Earn만) → 대체 툴로.
- 뉴스레터: 스티비 수수료 3.2%만(마진 좋음), 단 한국 시장 작음 + 별도 글쓰기.
- 자동화 대행: 단가 크나 영업 80%·고객 해지 리스크 → 성향 정면충돌, 비추천.
- 마이크로 SaaS/데이터 도구: 워크플로우 선택이 수익 좌우. "채널→제품" 수렴 종착점.

### 수익 가지도 (나무 구조)
- **뿌리·줄기**: AI 영상 채널 → ①콘텐츠·구독자 ②운영 데이터 축적.
- **1단계(부산물)**: 디지털 상품 + 제휴 (passive).
- **2단계(자산)**: 강의(자체) + (선택)뉴스레터.
- **3단계(수렴)**: 마이크로 SaaS·데이터 도구 ← 운영 데이터가 제품으로 순환.
- **가지치기**: 자동화 대행(제거).
- 핵심: 새 사업을 벌이지 않고 **채널 한 그루에서 가지를 친다.**

## 17. 전체 흐름 한눈에 + 핵심 원칙

**흐름**:
성향분석(분석형 빌더) → 도메인(콘텐츠) → 시드(AI 영상 생성) → 전략(채널 우선) → 콘셉트("AI로 만든다")·언어(한국어)
→ 리서치(수익화·정책·툴) → 벤치마크 → 파이프라인·OSS → 빈틈·편집툴·비용 → 주제(우주/주식) → 수익 가지도 → 로드맵·액션플랜.

**핵심 원칙 5가지**:
1. **내가 사용자인 영역만** (AI 의존 리스크 최소).
2. **채널 우선** — 제품이 평범해도 채널은 자산.
3. **목소리 해설 = 진정성·정책·신뢰**의 핵심 장치.
4. **검증 먼저** — 만들기 전에 팔리는지/통과 조건부터.
5. **벌이지 말고 가지 쳐라** — 채널 한 그루에서 수익이 뻗는다.
