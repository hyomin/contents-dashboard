개인 수익화 프로젝트:
'AI로 만든다'는 채널로
콘텐츠와 제품을 한 점에 모으다

분석형 빌더의 성향에서 출발해, 회사 업무(IIoT)와 분리된 사이드 프로젝트의 주제를 좁히고, 수익화·정책·툴을 리서치해 채널 우선 전략에 도달하기까지의 기록.

01 — 출발점

나는 어떤 빌더인가

분석형 빌더 · Data-Driven Builder

데이터·전략을 깊게 파서 직접 만드는 건 좋지만, 영업은 콘텐츠로 대신하고 싶고, 고객 응대 부담 없이 꾸준히 개선하며 안정 부수입을 쌓고 싶은 빌더.

동기 1·2데이터 분석 → 직접 제작
최하위고객 직접 응대
수익 스타일안정 부수입이면 만족
홍보콘텐츠로 자연스럽게
운영꾸준히 손보며 키우기
투입 시간주 5~10시간
02 — 확정된 결정

좁혀온 선택들

확정

시드 — AI 영상 생성 콘텐츠

코딩이 아니라, 생성형 AI로 영상을 만들어내는 그 과정·결과물이 콘텐츠. 만드는 대상이 곧 채널의 주인공.

확정

전략 — 채널 우선

제품 시장은 레드오션. 제품이 평범해도 채널은 자산으로 남는다. 콘텐츠로 키운 뒤 제품화.

확정

시그니처 — 'AI로 만든다' 콘셉트

주제는 자유롭게 열되, "이것도 AI로?"가 전 영상을 관통하는 후크. 다양성과 일관성을 동시에.

확정

노출 — 목소리 해설 (얼굴 X)

카메라·조명·실사편집 부담 제거. 동시에 진정성·정책·신뢰를 잡는 핵심 장치.

03 — 리서치 3대 결론

채널 설계 전에 알아낸 현실

수익화

포맷별 수익이 다르다

  • 주제별 RPM 격차 큼 — 고RPM(테크·교육) 섞기
  • 숏폼=유입, 롱폼=수익 기둥
  • 페이스리스 약점 → 제휴 + 목소리 기반 강의
  • 수익화까지 6~12개월 전제
정책

AI는 환영, 기만은 금지

  • 표시·비진정성은 별개 정책
  • 합성물은 떳떳이 표시 (불이익 없음)
  • 채널 전체 평가 — 동일 템플릿 반복 금지
  • 목소리 해설 = 비진정성 회피의 정답
툴

Higgsfield는 만능 아님

  • 시네마틱 비주얼엔 강함
  • 클립 10~20초 한계 → 롱폼은 이어붙이기
  • 캐릭터 드리프트 → '룩·콘셉트'로 묶기
  • 크레딧이 핵심 변동비
04 — 실행 로드맵

여기서부터 가는 길

STEP 1 · 지금

벤치마크 분석

채널 5개(한·영)를 같은 잣대로 비교 → 빈틈 찾기.

통과 조건 · "내가 들어갈 빈틈 + 차별 각도" 한 문장
STEP 2

채널 정체성 확정

채널명·콘셉트·룩·고정 포맷·타깃 5종 결정.

STEP 3

파이프라인 환경 세팅

화면 녹화·마이크·편집 도구 + 기존 자산 연결.

STEP 4

파일럿 3~5편

1번(결과물) 만들며 2번(과정) 동시 녹화 = 부산물.

STEP 5

발행 · 데이터 관찰

지표 관찰 → 분석형 강점 발휘 + 제품의 씨앗 데이터.

STEP 6

분기 결정

콘텐츠 더블다운 / 제품화 / 재실험 중 선택.

05 — 아직 안 정한 것

미결 의사결정 · 빈 파이프라인

  • 크레딧 예산 상한

  • 나레이션 환경 (마이크)
  • 화면 녹화 (2번 트랙)
  • 썸네일 시스템
  • 소재·실패 로그 체계
  • 성과 측정 체계
06 — 편집툴 직접 개발?

풀 편집기 ✕ · 자동화 ○

✕
풀 영상 편집기

타임라인·렌더링 엔진 = 수년짜리. 무료 대안 많음. 만들 이유 없음.

○
자동 조립 파이프라인

클립 stitching → 나레이션 → 자막 → 출력을 코드로. 10년 차 + Claude Code에 부합. (FFmpeg · MoviePy · Remotion · Whisper)

!
함정 — 도구부터 만들지 말 것

먼저 수동 3~5편 → 반복되는 지겨운 부분 식별 → 딱 그 부분만 자동화.

그 자동화 파이프라인 = 2번 트랙 콘텐츠 = 원래 원했던 제품. 채널·콘텐츠·제품이 한 점으로 수렴한다.
