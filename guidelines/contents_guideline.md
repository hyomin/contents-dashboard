## 공통 작성 원칙 (모든 포맷)

1. «OO 채널», «벤치마킹», «레퍼런스» 등 메타 표현 금지. 독자·시청자에게 직접 말하는 **발행용 톤**.
2. **구조·톤 레퍼런스**의 채널명·타 채널·영상·블로그 제목·표현은 본문에서 **완전히 제거**하거나 새 표현으로 바꿉니다.
3. **내용 레퍼런스**의 사실·수치·설명은 발행 주제에 맞게 **유지**하되, 출처·사이트명·URL·직접 인용 없이 **새 문장**으로 씁니다.
4. 표절·문장 복사 없이 사실·논지만 유지합니다.

## 블로그 발행 가이드 (writing / blog)

### 방향

블로그·카드뉴스·게시글은 «검색 의도에 답하는 제목»과 «스캔 가능한 본문 구조»가 핵심입니다. 참고 레퍼런스의 **제목·소제목 패턴만** 구조적으로 벤치마킹하고, 정보는 직접 조사한 사실로 채웁니다.

### 체크리스트 (Agent가 본문에 반영)

- 핵심 키워드 1개를 제목 앞 15자 안에 배치
- 첫 2문단 안에 결론·핵심 수치(요약) 제시 — «결론부터»
- H2 소제목 3~6개, 각 소제목 아래 3~5문장 또는 번호 목록
- 표·비교표·체크리스트로 정보 밀도 올리기 (체류 시간↑)
- 본문 중간 1곳에 내부 링크 또는 관련 글 언급
- 마지막 단락: 다음 행동(댓글·구독·저장·공유) 한 줄 CTA
- 레퍼런스 제목 패턴(숫자·질문·대비) 중 1개를 내 제목에 적용

### 제목 공식 (고성과 패턴)

- 숫자형: «2026년 ○○ 5가지» / «월 ○만원 절약하는 방법»
- 질문형: «왜 지금 ○○일까?» / «○○, 정말 효과 있을까?»
- 대비형: «○○ vs △△, 뭐가 나을까» / «전문가 vs 초보 차이»
- 시한성: «올해 말 전에» / «○월부터 바뀌는» — 트렌드 키워드와 결합

### 인트로 패턴 (택1 — 글 성격에 맞게)

- **질문·공감형 (기본값):** 독자의 궁금증·고민을 1문장 질문으로 제시 → «오늘은 ~을 정리합니다»로 글의 구조를 미리 안내. (예: «○○ 목표가 90만 원, 너무 과한 숫자일까요? 오늘은 그 근거를 숫자로 나눠보겠습니다»)
- **고정 자기소개형:** 매 글 동일한 한 문장 자기소개로 시작해 브랜드 일관성 확보. 시리즈·정기 발행 콘텐츠에 적합.
- **근황·일기형:** «오랜만에 글을 올리네요» 식 근황 토크로 시작. 비정기 발행·개인 일기형 콘텐츠에 적합.

### 글 유형별 구성 템플릿 (택1 — 주제에 맞는 유형 선택)

**A. 시황/매크로형** (시장 전반·경제 이슈·정책 해설)
1. 한 줄 핵심 요약(굵게) → 한두 문장 단위로 짧게 끊어 본문 전개
2. `-소제목-` 또는 굵은 텍스트로 주제 전환 표시 (헤딩 태그 없어도 가독성 확보 가능)
3. 관련 표·이미지 가이드 1~2개 삽입 (`agent:blog-image`)
4. «수혜주는?», «참고하면 좋은 글» 같은 다음 행동 제안으로 마무리

**B. 종목/주제분석형** (특정 종목·기업·서비스·이슈 심층 분석)
1. `📌 목차` — H2 4개 이상일 때 본문 최상단에 배치
2. 번호형 H2: 개요·이슈 → 변동/이슈의 근거(FACT) → 비교표(경쟁사·전후 비교) → 활용·투자 포인트 또는 리스크
3. `한 줄 요약` 박스 — 핵심 결론을 3줄 이내로 정리
4. `FAQ` 2~5문항 — 검색 롱테일 대응
5. 관련 글 추천 1개 (내부 링크 멘션)

**C. 재테크/정보형** (제도·금융상품·생활 정보)
1. 이모지 질문형 인트로로 공감 유도
2. ✅ 또는 번호+이모지 H2로 핵심 항목 나열, 항목별 비교표 1개 이상
3. «체크포인트»/«전문가 조언» 식 반복 박스로 일관성 부여 (선택)
4. `FAQ` + (제휴/면책 고지가 필요한 주제라면 최상단 또는 최하단에 명시)

### 네이버 블로그 (원본/기준)

- 썸네일·대표 이미지: 제목과 같은 메시지, 텍스트 3~5단어 이내
- 첫 H2 전 150~200자: 검색 의도에 대한 직접 답변 (AI 요약·스니펫 대비)
- 키워드 반복보다 동의어·관련어·LSI 키워드 자연 배치
- 목차: H2 4개 이상이면 자동 목차 활성화

### 3플랫폼 동시 발행 구조 (네이버 → 티스토리 → Google Blogger)

콘텐츠는 **네이버 블로그를 원본**으로 작성하고, 동일한 본문 구조(H2·FAQ·이미지 가이드)는 그대로 둔 채 **제목·메타·태그/라벨·첫 단락만** 플랫폼별로 조정해 티스토리·Blogger에 미러 발행합니다. 본문 문장이 100% 동일해도 무방하나, 검색 스니펫 차별화를 위해 제목·메타·첫 단락은 플랫폼별로 살짝 다르게 씁니다.

**발행 순서 및 자동화 가능 여부**

| 플랫폼 | 발행 순서 | 자동화 |
|---|---|---|
| 네이버 블로그 | ① 원본 선발행 | 공식 포스팅 API 없음 — 수동 게시 |
| 티스토리 | ② 본문 미러 + 카테고리·태그만 조정 | 공식 Open API 신규 등록 중단 — 수동 게시 |
| Google Blogger | ③ 본문 미러 + 제목·메타·라벨만 조정 | Blogger API v3 존재 — OAuth 연동 시 자동발행 가능(별도 작업) |

**플랫폼별 메타 변형 규칙**

| 항목 | 네이버 (원본) | 티스토리 | Google Blogger |
|---|---|---|---|
| 제목 | 핵심 키워드 앞 15자, 숫자·질문·대비형 공식 | 원본과 동일 또는 소폭 변형 | 핵심 키워드 앞 40자, 이모지·특수문자 지양(SERP 가독성) |
| 메타 설명 | 노출 비중 낮음 — 본문 첫 150~200자가 사실상 메타 | 120자 이내 | 140~160자, 첫 문장에 키워드 1개 + «이 글에서 얻는 것» 한 줄 |
| 태그/카테고리/라벨 | 자유 태그 | 카테고리 1개 + 태그 3~5개 | 라벨 3~5개 (주제·세부키워드·시한성 「2026」 혼합) |
| 대표 이미지 | 텍스트 3~5단어 이내 | 네이버와 동일 이미지 재사용 가능 | 1200px 권장 + alt 텍스트에 키워드 자연 포함 |
| 첫 단락 | 검색 의도 직접 답변 150~200자 | 네이버와 동일 가능 | 150자 안에 결론 요약 (Featured snippet 대비) |

**Google Blogger 추가 규칙**

- AdSense: 본문 1,000자 이상 권장, 제목·H2 사이 광고 과밀 배치 금지, «클릭 유도» 문구 본문에 넣지 않음
- 내부 링크: 관련 글 1~2개, 외부 링크는 신뢰 출처만
- 영문 확장(선택): 동일 주제 «영문 제목 1줄 + 영문 요약 2문장»을 글 맨 아래 별도 블록으로 — 글로벌 검색 실험용

**티스토리 추가 규칙**

- 카테고리는 기존 운영 카테고리 중 주제와 가장 가까운 1개 선택 (신규 카테고리 생성 지양)
- 태그는 검색 유입용 롱테일 키워드 위주로 3~5개
- 본문 내 광고(애드센스·애드핏) 배치 시 H2 사이 과밀 금지 (Blogger와 동일 원칙)

### 본문 구조 템플릿 (기본형 — 위 유형별 템플릿이 맞지 않을 때)

- 도입(10%): 문제 제기 + «이 글에서 얻는 것» 1문장
- 본론(75%): H2별 «주장 1줄 → 근거·사례 → 실천 팁»
- 결론(15%): `한 줄 요약` 박스(3줄 이내) + CTA + (선택) FAQ 2~3개
- `📌 목차`: H2 4개 이상이면 본문 최상단에 직접 작성 (네이버 자동 목차와 별개)
- FAQ: «○○란?» «언제 해야 하나?» — 검색 롱테일 대응

### SNS 카피 (캡션·카드뉴스)

- 1문장 후킹(질문·숫자·반전) → 2~3문장 가치 → CTA
- 해시태그: 플랫폼 규칙에 맞게 3~7개
- 카드뉴스: 1장=1메시지, 마지막 장에 저장·공유 유도

### 마무리 팁

참고 레퍼런스에서 vs.Avg가 높은 글 3개의 «제목 구조»와 «첫 H2 소제목»만 추출해 표로 정리한 뒤, 내 주제에 맞게 치환합니다. 문장 전체 복사는 피합니다.

## 주식 리포트 차트 이미지 가이드 (fullContent에 삽입)

"환기용 이미지 가이드(📷 직접 제작·삽입)"와 "표 가이드(📊 표 가이드 — 직접 제작·삽입)"는 **사용하지 않습니다**.

이 가이드 블록은 **사실 기반**으로 작성합니다 — 최근 추세(가격·이동평균), 일일 등락 흐름(거래량·등락률), 향후 추정 구간(추세선) 등 아래 "이번 리포트에서 사용 가능한 차트" 목록의 실제 데이터 특징을 근거로 해설합니다.

## 주식 리포트 (writing / stock-report 모드)

### 방향

한국/미국 증시 시세 데이터를 바탕으로 발행하는 **주식 리포트**입니다. `agent:blog`의 3플랫폼 발행 구조·SEO 규칙을 그대로 따르되, 아래 3개 모드별 구성 순서를 사용합니다. 모든 모드는 `agent:stock-chart-image`의 차트 이미지 가이드를 동반합니다.

### 종목/지수별 일일 리포트 모드

워치리스트에 등록된 종목·지수 **각각에 대해 개별로 1건씩** 발행하는 일일 리포트입니다 (전체를 묶은 통합 시황 요약이 아닙니다).

1. **오늘의 시세·등락 요약** — 해당 종목/지수의 오늘 종가·등락률을 2~3문장으로 요약. 지수(코스피·코스닥·S&P500·나스닥 등)인 경우 시장 전체 흐름 중심으로, 개별 종목인 경우 해당 종목 흐름 + 소속 시장(코스피/코스닥/나스닥 등) 동향을 한 줄 덧붙여 설명
2. **최근 시세 패턴 분석** — 최근 N일(N=10) 시가·종가·등락률 표를 바탕으로 현재 흐름과 과거 유사 구간을 비교 설명 (표 형식 권장, 열 구성: 날짜 | 시가 | 종가 | 전일대비 등락률)
3. **공개정보·예정 이벤트 큐레이션** — 제공된 경제 뉴스/토픽 중 해당 종목·시장과 관련된 항목을 인용 없이 재구성해 소개 (실적 발표 일정, 산업 동향 등)
4. **고정 디스클레이머** — 본문 마지막에 아래 문구를 **그대로** 포함 (자본시장법 — 단순 정보 제공, 투자 권유 아님)

> ※ 본 콘텐츠는 투자 판단에 참고할 수 있는 시장 정보·데이터 분석을 제공하기 위한 목적으로 작성되었으며, 특정 종목의 매수·매도 시점이나 투자를 권유하는 내용이 아닙니다. 투자에 대한 최종 결정과 책임은 투자자 본인에게 있습니다.

제목 형식: `"YYYY-MM-DD ○○ 일일 리포트"` (○○은 해당 종목/지수명).

### 관심 종목 분석 모드 (포커스 리포트)

사용자가 직접 입력한 1~3개 종목(워치리스트와 무관)을 즉석 시세 데이터로 분석하는 모드입니다. `agent:blog`의 3플랫폼 발행 구조·SEO 규칙, 위의 고정 디스클레이머는 동일하게 적용하며, 본문 구성은 아래 순서를 사용합니다.

1. **종목 개요·최근 이슈** — 분석 대상 종목 소개 + 사용자가 제공한 참고 메모(있는 경우)를 반영한 최근 이슈 요약
2. **가격 추이·패턴 분석** — 최근 시세 데이터(시가·종가·등락률)를 표로 정리(열 구성: 날짜 | 시가 | 종가 | 전일대비 등락률)하고 흐름·과거 유사 구간을 비교 설명
3. **업종/거시 맥락** — 제공된 경제 뉴스/토픽 중 관련 있는 항목을 인용 없이 재구성해 업종·시장 전반 맥락 제공
4. **고정 디스클레이머** — 일일 리포트 모드와 동일 문구를 본문 마지막에 그대로 포함

제목 형식: `"YYYY-MM-DD ○○ 종목 분석 리포트"` (○○은 분석 대상 종목명, 여러 개면 "·"로 구분).

### 섹터/카테고리 종합분석 모드

기업명 대신 "반도체주", "방산주" 같은 **섹터/카테고리**를 선택해 해당 섹터 구성종목들을 종합 분석하는 모드입니다. `agent:blog`의 3플랫폼 발행 구조·SEO 규칙, 위의 고정 디스클레이머는 동일하게 적용하며, 본문 구성은 아래 순서를 사용합니다.

1. **섹터 개요·최근 화두** — 선택된 섹터의 산업 개요 + 제공된 경제 뉴스/토픽 중 해당 섹터와 관련된 최근 화두 요약
2. **구성종목별 가격추이 비교표** — 구성종목별 최근 시세 데이터(시가·종가·등락률)를 한 표에 정리(열 구성: 날짜 | 시가 | 종가 | 전일대비 등락률)해 종목 간 흐름 차이를 비교 설명
3. **섹터 공통 거시/이벤트 맥락** — 제공된 경제 뉴스/토픽 중 섹터 전반에 영향을 주는 거시 이슈·예정 이벤트를 인용 없이 재구성해 소개
4. **고정 디스클레이머** — 일일 리포트 모드와 동일 문구를 본문 마지막에 그대로 포함

제목 형식: `"YYYY-MM-DD ○○(섹터) 산업 분석 리포트"` (○○은 섹터/카테고리명).

### 작성 시 주의사항 (공통)

- "지금 사세요", "매도 타이밍입니다" 등 **매수·매도 시점을 직접 권유하는 표현 금지**
- 수치는 제공된 데이터 범위 내에서만 서술하고, 임의로 추정한 미래 가격을 단정적으로 제시하지 않음
- 제목에는 날짜(YYYY-MM-DD 또는 "○월 ○일")와 "일일 리포트"/"종목 분석 리포트"/"산업 분석 리포트" 등 핵심 키워드 포함

## 숏폼 공통 가이드 (video / shortform)

### 방향

롱폼은 구조·신뢰, 숏폼은 **첫 1초 훅**이 전부입니다. 트렌드 주제라면 «왜 지금인가»를 도입 **0~5초** 안에 말합니다.

### 체크리스트 (Agent가 스크립트에 반영)

- 0~3초: 질문 또는 결과 미리보기(숫자/반전) — 아래 «바이럴 제목·썸네일»에서 던진 떡밥을 바로 회수
- 본론 전 «이 영상에서 얻는 것» 한 문장 예고
- 텍스트 훅(숏)으로 이탈 구간 막기
- 엔딩: 구독·다음 영상 예고 + 동일 키워드 자막 반복
- 루프 가능한 구조(마지막이 첫 장면과 연결) 실험
- 자막은 키워드만 굵게, 한 줄 길이 짧게

### 바이럴 제목·썸네일 설계 (스크립트보다 먼저 확정 — 클릭률을 좌우)

스크립트를 쓰기 전에 제목·썸네일을 먼저 설계합니다. **둘의 역할 분리와 시너지**가 클릭률(CTR)의 핵심입니다.

- **역할 분리 (텍스트 반복 절대 금지):** 썸네일 = "질문·호기심" 던지기 / 제목 = "해답 힌트·맥락" 제공. 같은 단어를 양쪽에 쓰지 않습니다.
  - 예) 썸네일 텍스트 «왕의 진짜 얼굴» + 제목 «태종 이방원이 밤마다 몰래 했던 행동 3가지»
- **제목 공식 (1~2개 결합, 15~25자):** FOMO(놓치면 손해) · 호기심 갭(다 알려주지 않기) · 홀수 법칙(3·5·7가지) · 뾰족한 타겟팅(예: «직장인 필독»)
- **썸네일 텍스트:** 5~7자 이내 극단적 미니멀. 가독성 치트키 — 노랑 텍스트+검정 테두리 또는 빨강 배경+흰색 텍스트. 감정(놀람·반전·분노)이 담긴 «하이라이트 프레임» 1장만 사용
- **떡밥 회수:** 영상 시작 **0~3초 안에** 썸네일이 던진 질문의 단서를 즉시 제시해 이탈을 막습니다.

#### 발행 전 CTR 체크리스트 (fullContent 맨 아래에 포함)

- [ ] 썸네일 텍스트가 5~7자 이내로 한눈에 읽히는가?
- [ ] 썸네일 텍스트와 제목이 서로 반복되지 않고 «질문 ↔ 해답» 시너지를 내는가?
- [ ] 타겟 시청자가 제목을 보고 «어? 내 얘긴데?» 할 수 있는가?
- [ ] 영상 시작 0~3초 안에 썸네일의 떡밥을 바로 회수하는가?

### 플랫폼 (업로드)

- YouTube Shorts / Reels / TikTok
- 세로 **9:16**, **60초 이내** (클립 여러 개 → 캡컷 등에서 이어 붙임)
- 첫 **1~2초 훅** · 루프 가능 엔딩 · 짧은 자막(onScreenText) 필수

### 숏폼 장면 스크립트 (fullContent 필수)

- **45~60초** 기준, **3~5개 장면**으로 분할합니다.
- 레퍼런스 흔적 제거·발행용 톤 유지, 본문은 **시간대별 장면 블록**이 핵심입니다.
- 영상 클립은 직접 촬영·OBS 화면 녹화·스톡 영상으로 준비 후 캡컷 등에서 편집합니다.

#### fullContent 장면 블록 형식

```
[0~5초] 장면1 · 첫 훅) 오늘 작업에서 이런 오류가 나왔어요.
**화면 설명:** 화면 녹화 클립 — 오류 상황 클로즈업
**자막:** 이게 왜 이러는 거지...

[6~14초] 장면2) ...
```

#### fullContent 맨 아래 필수 섹션

## 📱 자막 오버레이
- (장면별 핵심 자막 1줄씩, 4~6개)

## 🎬 편집 메모
- 도구: 캡컷 (또는 CapCut) · 세로 9:16 · 총 N씬
- 클립 소스: (직접 촬영 / OBS 화면 녹화 / 스톡 영상)
- 편집: 클립 시간순 concat → 자막 삽입 → Shorts 업로드

#### 규칙

- 시간대는 **0초부터 연속**, 겹치지 않게, 합계 **60초 이내**
- 장면 제목은 **한글**로 «무슨 장면인지» 바로 알 수 있게
- 화면 설명은 촬영·편집 방향 안내용 (AI 생성 프롬프트 아님)

### 마무리 팁

벤치마킹으로 등록한 고성과 영상 3개의 **도입부 문장 구조**만 참고한 뒤, 자신의 정보로 치환합니다.
